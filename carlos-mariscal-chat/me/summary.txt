## Key Projects — Detailed Case Studies

### 1. AI Support Assistant (AIDA) — Intuit (2023–2025)
Problem: Intuit's internal developer support was overwhelmed across hundreds of Slack channels. No automated triage or response. Knowledge scattered across Slack history, wikis, Autura docs, GitHub markup, StackOverflow.
Architecture: Multi-service platform — Slack App (Lambda + Go) for event routing, AIDA Service (Java, Spring Boot, K8s) for orchestration with OpenAI and Glean RAG, Support Eco Service (Go) for StackOverflow-style Q&A, Insights Pipeline (Python, Lambda, SQS, DynamoDB) using clustering models and specialized prompts to identify doc gaps/enhancements/tool opportunities, Metrics Dashboard (Java, Spring Boot, DynamoDB), and a dedicated Retry Architecture for transient failures.
Outcomes: 220+ channels served, 85% message traffic handled automatically, 27% support volume reduction, data-driven insights pipeline, metrics dashboard for continuous improvement.

### 2. Sanctions Screening Platform Migration — Intuit (2021–2023)
Problem: Needed to migrate from Amber Road to LexisNexis after acquiring Credit Karma (130M+ users) and Mailchimp (14M+ users). Legacy platform lacked reporting, batch screening, and audit capabilities.
Architecture: Event-driven platform with Real-Time Screening API calling Bridger/LexisNexis, Kafka topics (SanctionScreeningTopic, SanctionScreeningEnvelopeTopic, ScreeningEventTopic) with dedicated consumers, multi-stage Batch Processing Pipeline with SFTP/S3, Audit Data Warehouse storing Screening/Contact/Decision/Alert History, Data Migration from Amber Road, CSV Ingestion for OM Agent uploads, and resilience via API retry and Screening Event retry with exhaustion handling.
Outcomes: 130M+ Credit Karma records screened, 14M+ Mailchimp records screened, delivered reporting/batch/audit capabilities, supported post-acquisition compliance across all Intuit product teams.

### 3. AI Performance Review Assistant — Zoox (2025–Present)
Problem: Biannual performance reviews for 2,800+ employees consumed thousands of manager hours with no consistency.
Architecture: AI-powered pipeline with automated feedback collection, LLM-based analysis for structured reviews, parallelism and batching for throughput, intelligent caching to reduce redundant LLM calls, and automated role-based access control.
Outcomes: 26,000+ review artifacts generated, 2,800+ employees covered, ~1,400 manager hours saved per cycle, 60–80% AI cost reduction via caching.

### 4. Subscription Signup Idempotency — Intuit (2019–2021)
Problem: Critical production bugs from duplicate signup requests — multiple subscriptions created, grants inactivated, incorrect correlation IDs, poison messages unprocessed. Directly impacted QBO subscription success rates.
Architecture: Event-driven idempotency solution using DocumentDB for deduplication keyed by webs_request_id, 7+ async consumers (Signup, Process Subscription, Caching, Event Publisher, Failure Event, Transaction History, Signup Retry) each publishing typed SignupEventDTOs, event sourcing for full audit trail, automatic STANDBY replay on failure, Tracking UI and API for L1/L2 support with replay capability, poison message handling with configurable exhaustion, and data lifecycle cron (3/7/30 day purges).
Outcomes: Eliminated duplicate subscription bugs, real-time signup lifecycle visibility, automated failed transaction replay, mentored team of 5 engineers in design patterns and distributed systems.

### 5. Private Catalog Offers Refactoring — Intuit (2017–2019)
Problem: Monolithic 1,381-line OfferServiceImpl class. VisualVM showed 1,236 DB calls taking 53.9s. Production worst-case response: 43,485ms. 20-minute unit test runs.
Architecture: Strategy pattern for query actions (Addition, Migration, Change, Switch strategies), extracted Catalog Model Library for reuse, Redis/CouchBase caching with Quartz-scheduled refresh (2h/12h), concurrent processing for parallelized DB calls, early pagination (before processing, max 500), single batched query replacing 1,236 individual transition calls.
Outcomes: 60% performance gains, 90% faster response times with caching, DB calls reduced from 1,236 to 1, SOLID decomposition, migrated from Mule to Spring Boot on AWS.
